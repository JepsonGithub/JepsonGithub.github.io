什么是Vue.js

v-if 和 v-show

区别在于:

  1. v-if: 如果值为 false, 内部元素直接不渲染, 切换flag时, 会重复删除节点, 解析渲染节点
  2. v-show: 如果值为 false, 只是让节点的 display 为 none, 切换时, 切换的 display

应用:
所以在一些需要频繁切换显示状态的, 比如说 tab 栏切换, 弹出层等等, 用 v-show 比较合适, 频繁切换也不会太耗性能

而形如页面模板渲染, 不需要频繁切换的, 有就有没有就没有的, 就用 v-if 比较合适, 有些内容, 如果不想渲染, 还出现在页面节点中就不太好, 直接不渲染就好了

v-on 指令的使用

注册事件的时候, 可以直接写函数名称, 也可以加上() 当成表达式调用, 但是区别在于, 写成函数调用的那种是可以传参的, 这是比较方便的

另外: v-on:事件 可以简写为 @事件=”事件处理函数()”

在 vue 事件中获取事件对象

  1. 如果事件是通过 @click=函数名 函数名来注册的, 直接通过 e 就可以拿到事件对象
  2. 但是如果是 @click=函数名() 表达式注册的, 由于是手动调用, 所有就没有默认的 event 参数, 如果需要事件对象必须手动传 $event

vue 中的事件修饰符

事件修饰符:
通过事件修饰符, 我们可以为事件处理函数指定一些特定的功能

  1. 阻止冒泡 stop
  2. 阻止默认行为 prevent
  3. 捕获阶段触发 capture
  4. 只有自己能够触发 self, (即使冒泡了也不触发)
  5. 只触发一次 once

v-for 列表渲染

基本语法: v-for="(item, key) in friends" item 键, key 值

key 属性的使用

官方api: https://cn.vuejs.org/v2/api/#key

key 的特殊属性主要用在 Vue 的虚拟 DOM 算法，在新旧 nodes 对比时辨识 VNodes。
如果不使用 key，Vue 会使用一种最大限度减少动态元素, 并且尽可能的尝试修复/再利用相同类型元素的算法。
使用 key，它会基于 key 的变化重新排列元素顺序，并且会移除 key 不存在的元素。

可能很多人会觉得好绕好难懂, 把我下面准备好的 demo 跑一下, 看下有 :key 和 没有 :key 的区别, 才会真正理解 key 属性的作用

不加:key, 新旧node没有一一对应, Vue会最小限度动态元素, 所以只会改变内容, 也就是只改里面的文本, 会出现错位的 bug 等

而如果加上:key了以后, Vue 会基于 key 的变化, 一一对应起来

tips: 所以如果内容节点需要动态变化的, 就加上 :key, 一般渲染完成就不变的, 就不用加 :key
