什么是Vue.js

  • Vue.js 是前端的主流框架之一，和Angular.js、React.js 一起，并称为前端三大主流框架！
  • Vue.js 是一套构建用户界面的框架，只关注视图层，它不仅易于上手，还便于与第三方库或既有项目整合。（Vue有配套的第三方类库，可以整合起来做大型项目的开发）
  • 前端的主要工作？主要负责MVC中的V这一层；主要工作就是和界面打交道；

Vue.js 初体验

vuejs 的强项就在于数据绑定

  1. 双向数据绑定 (一般用于表单元素)
    当数据和元素进行双向数据绑定之后, 数据发生变化, 元素中的内容也会发生变化, 元素中的内容发生变化, 数据也会发生变化
  2. 单向数据绑定 (一般用于展示型元素)
    当数据和元素进行单向数据绑定之后, 数据发生变化, 元素中的内容也会发生变化, 元素的内容发生变化, 不会影响到数据

Vue 双向数据绑定基本原理

前置知识点: defineProperty

Object.defineProperty
功能: 可以用来给对象添加新属性, 也可以修改对象已经存在的属性
特点: 这种方式添加的属性可以设置属性的特征

属性的特征:

  1. 属性可不可写 writeable
  2. 属性可不可遍历(枚举) enumerable
  3. 属性可不可删除, 配置 configurable
  4. 属性的默认值 value
  5. 属性的设置和获取方法( 重点 )

语法: Object.defineProperty(要添加属性的对象, 要添加的属性名, 属性的特征对象)

get 和 set(重点)

注意: value 和 writable 属性 和 get, set 不能共存, 共存会报错

  • get 函数, 属性在被获取值时, 自动调用, 获取的值是 get 函数的返回值
  • set 函数在 属性被赋值时, 自动调用

可以在set方法中监听属性的变化
也可以在set方法中, 为属性的赋值设置合法性校验

双向数据绑定

利用 es5 提供的defineProperty, 可以实现数据的双向数据绑定

我们自己可以监听 input 框里面的变化, 可以同步到数据对象, 而数据对象的变化, 我们一般不好监听, 但是利用 es5 提供的 defineProperty, 通过监听get, set 即可实现数据的双向绑定

v-bind 使用 vue 给属性绑定数据

当需要给html标签中的属性绑定一个数据的时候, 不能使用插值表达式,
要用 v-bind 指令

  • 语法 v-bind:属性名=”数据名称
  • 简写 :属性名=”数据名称”

注意: 如果视图中用到的属性, 在数据模型中没有定义, 就会报错

v-if 和 v-show

区别在于:

  1. v-if: 如果值为 false, 内部元素直接不渲染, 切换flag时, 会重复删除节点, 解析渲染节点
  2. v-show: 如果值为 false, 只是让节点的 display 为 none, 切换时, 切换的 display

应用:
所以在一些需要频繁切换显示状态的, 比如说 tab 栏切换, 弹出层等等, 用 v-show 比较合适, 频繁切换也不会太耗性能

而形如页面模板渲染, 不需要频繁切换的, 有就有没有就没有的, 就用 v-if 比较合适, 有些内容, 如果不想渲染, 还出现在页面节点中就不太好, 直接不渲染就好了

v-on 指令的使用

注册事件的时候, 可以直接写函数名称, 也可以加上() 当成表达式调用, 但是区别在于, 写成函数调用的那种是可以传参的, 这是比较方便的

另外: v-on:事件 可以简写为 @事件=”事件处理函数()”

在 vue 事件中获取事件对象

  1. 如果事件是通过 @click=函数名 函数名来注册的, 直接通过 e 就可以拿到事件对象
  2. 但是如果是 @click=函数名() 表达式注册的, 由于是手动调用, 所有就没有默认的 event 参数, 如果需要事件对象必须手动传 $event

vue 中的事件修饰符

事件修饰符:
通过事件修饰符, 我们可以为事件处理函数指定一些特定的功能

  1. 阻止冒泡 stop
  2. 阻止默认行为 prevent
  3. 捕获阶段触发 capture
  4. 只有自己能够触发 self, (即使冒泡了也不触发)
  5. 只触发一次 once

默认click事件是会冒泡的, a有默认跳转行为, 我们可以通过 e.preventDefault e.stopPropagation 阻止冒泡, 阻止默认行为, 这在 vue中是一样的

但是在 vue 中, return false 不能阻止冒泡 和默认行为, 需要注意!!!

key 属性的使用

官方api: https://cn.vuejs.org/v2/api/#key

key 的特殊属性主要用在 Vue 的虚拟 DOM 算法，在新旧 nodes 对比时辨识 VNodes。
如果不使用 key，Vue 会使用一种最大限度减少动态元素, 并且尽可能的尝试修复/再利用相同类型元素的算法。
使用 key，它会基于 key 的变化重新排列元素顺序，并且会移除 key 不存在的元素。

可能很多人会觉得好难懂, 那就把我下面准备好的 demo 跑一下, 看下有 :key 和 没有 :key 的区别, 才会真正理解 key 属性的作用

不加:key, 新旧node没有一一对应, Vue会最小限度动态元素, 所以只会改变内容, 也就是只改里面的文本, 而如果加上:key了以后, Vue 会基于 key 的变化, 一一对应起来

tips: 所以如果内容节点需要动态变化的, 就加上 :key, 一般渲染完成就不变的, 就不用加 :key

Vue-resource 和 axios 的区别

  1. vue-resource 是一个vue插件, 依赖于 vue, 而 axios 是一个单独的工具包, 在哪都能用, 专门用来发送 ajax 请求
  2. vue-resource 支持 jsonp, axios 不支持 jsonp ( 因为axios作者认为, 跨域问题就应该由服务器端通过配置请求头, 或者配置服务器代理解决, 而不是使用 jsonp, 加大前后端开发维护成本 )

axios 发送请求, 默认 data 是 “name=Jepson” 这种 url-encoded 格式的, 但是我们一般习惯还是传对象, 所以就需要配置请求拦截器

计算属性

计算属性: 计算属性的本质就是一个函数, 这个函数会有一个返回值, 使用这个函数的时候, 可以直接像使用属性一样方式去使用它, 展示出来的内容就是这个函数的返回值

计算属性的基本说明 (计算器案例)

什么时候用计算属性? 什么时候用methods呢?

  1. 当需要使用一个数据, 而这个数据是根据其他数据出来的, 那么这个数据就要使用计算属性来做
  2. methods是将一些功能代码进行封装! 事件处理函数, 可以复用的一些请求函数…

计算属性和methods的区别

  1. 计算属性: 有缓存, 如果计算过一次之后, 数据没有发生改变, 那么下次来使用这个计算属性, 将直接从缓存中获取, 而不是重新计算, 这样可以提高程序运行的效率!
  2. methods 中声明函数: 虽然也能实现计算属性类型的功能, 但是他没有缓存,每次调用都会重新将函数的代码执行一遍, 效率会比较差

单页应用程序

单页应用程序基本概念

  1. 什么是单页应用(SPA)
    整个网站, 只有一个页面的这种网站就是所谓的单页应用
    和他对应的就是多页应用程序(平时我们所写的项目基本都是多页应用程序)

  2. 单页应用程序的基本原理
    将所有的功能全都放在同一个页面中, 当进行连接跳转的时候, 只需要将不同的内容进行展示即可
    不需要将整个页面重新加载

  1. 单页面应用的好处
    (1) 减少了浏览器向服务器发送的请求的数量
    (2) 提升了用户体验

存在的问题: 首屏加载时间较长, 不利于 SEO 优化
