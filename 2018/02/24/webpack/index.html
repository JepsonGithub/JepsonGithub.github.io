webpack的基本使用

标签：webpack

webpack 是一个模块化打包工具!
webpack 会分析模块之间的依赖关系, 并且将这些模块解析生成代表这些模块的静态资源

webpack 安装

  1. 全局安装

当做完全局安装后, 在命令行中就可以使用 webpack 了

  1. 本地安装(官方推荐)

本地安装, 本地安装后, 在全局不能使用, 但是在npm脚本中是可以使用的, 因为执行 package.json 配置的脚本时, 会自动将 node_modules 临时加到环境变量中去, 这样是可以访问到的

webpack 支持的模块化标准

  1. AMD (requirejs)
  2. CommonJS (node)
  3. ES6 模块化

webpack 的 基本语法

webpack main.js build.js
入口文件 main.js 自动加载依赖
输出文件 build.js

使用 webpack.config.js 进行命令的简化处理

webpack.config.js 默认配置文件，webpack会自动去找这个文件

之后直接执行 webpack 就行了, webpack 会自己去找到 webpack.config.js 中的内容进行相应的打包工作

webpack 打包 css， (style-loader,css-loader)

导入 css 和 导入 js 一样

但是webpack 默认只支持js模块的解析，所以其他文件类型的模块需要加载依赖, 进行加载解析

  • style-loader
  • css-loader

cnpm init 初始化一个 package.json文件
cnpm install style-loader css-loader --save-dev 利用package.json 进行管理依赖

下次可以直接利用 cnpm install进行依赖安装

注意： 在webpack.config.js 中配置下style-loader 和 css-loader，且有顺序要求，必须是先 style-loader 再 css-loader

webpack.config.js

webpack 打包 less (sass等同理)

步骤：

  1. 安装： cnpm install less-loader less --save-dev， style-loader, css-loader之前已经安装过了，也是必须的
  2. 在webpack.config.js 中添加 less-loader

利用 webpack 来打包图片

webpack 打包将图片转成 base64 会将图片所占大小变大1/3左右, 比如原来图片20kb, 转成base64, 近30kb, 所以小图片转成 base64比较合适, 但是大图片还是直接通过路径引入比较合适
步骤：

  1. 安装依赖 url-loader, file-loader, cnpm install url-loader file-loader --save-dev
  2. 将url-loader, file-loader 配置到 webpack.config.js 文件中

webpack.config.js

利用 url-loader 完成字体图标的解析

在loader中加入 ttf|woff2|woff|eot 文件的解析，url-loader也可以解析 ttf|woff2|woff|eot 等文件，需要其他文件可以自行添加扩展名

利用 babel-loader等包实现 es6 转 es5

步骤：

  1. 安装相应依赖包

    cnpm install babel-core –save-dev
    cnpm install babel-loader –save-dev
    cnpm install babel-preset-env –save-dev （转码器）

  2. 在目录新建.babelrc 文件

  3. 在webpack.config.js中配置loader

html-webpack-plugin 实现页面刷新和自动打包

https://www.npmjs.com/package/html-webpack-plugin
作用: 自动将 src 目录中 index.html 复制到dist目录中, 并且自动为其添加 编译输出文件的 引用

  1. 安装
    cnpm install html-webpack-plugin -D
  2. 配置 webpack.config.js

webpack-dev-server 热更新

webpack-dev-server 是跑在内存里面的
步骤：

  1. 安装： cnpm install webpack-dev-server --save-dev
  2. webpack-dev-server --hot --open --port 3008
    （热编译 –hot, 端口打开在 3008 ）
  3. 这个指令这么长，想缩短，就在package.json配置

配置完成之后 npm run dev 就可以运行了

webpack-dev-server 常用功能配置

  1. –port 用来指定服务器端口号
    webpack-dev-server –port 端口号

  2. –contentBase
    webpack-dev-server –port 9999 –contentBase ./

  3. –hot 用来配置 webpack-dev-server 的热更新, 热加载功能, 提高效率

mac 下默认打开的 safari, 配置chrome 打开
